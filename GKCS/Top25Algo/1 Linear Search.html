Linear Search

Category: Searching (Green)

Introduction: Linear Search, also known as Sequential Search, is the simplest searching algorithm. It works by sequentially checking each element in a list until the target element is found or the entire list has been searched. Think of it like looking for a book on a shelf by checking each book one by one from left to right. While not the most efficient for large datasets, it's easy to understand and implement, and works on both sorted and unsorted data.

Working Principle: The algorithm works by iterating through each element in the collection:

  1. Start from the first element of the array/list.
  2. Compare the current element with the target value.
  3. If they match, return the current index.
  4. If they don't match, move to the next element.
  5. Repeat steps 2-4 until the element is found or the end of the list is reached.
  6. If the end is reached without finding the element, return a "not found" indicator (usually -1).
Pseudocode:
function linear_search(arr, target) is
  for i from 0 to length(arr)-1 do
    if arr[i] == target then
      return i
  return -1 // Target not found

Example & Dry Run:
Let's search for the value 16 in the array: [5, 2, 8, 16, 23, 12, 38, 56, 72, 91].

  • Step 1: Check index 0: arr[0]=5. 5 ≠ 16, so continue.
  • Step 2: Check index 1: arr[1]=2. 2 ≠ 16, so continue.
  • Step 3: Check index 2: arr[2]=8. 8 ≠ 16, so continue.
  • Step 4: Check index 3: arr[3]=16. 16 = 16, match found! Return index 3.
The target was found in 4 steps. In the worst case (searching for 91), it would take 10 steps.

Time and Space Complexity:

  • Best Case: O(1) - The target is the first element.
  • Average Case: O(n) - On average, we need to check half the elements.
  • Worst Case: O(n) - The target is the last element or not in the list.
  • Space Complexity: O(1) - It uses a constant amount of extra space.

Implementation in Multiple Languages:

// C Code Example
#include <stdio.h>

int linearSearch(int arr[], int size, int target) {
    for (int i = 0; i < size; i++) {
        if (arr[i] == target) {
            return i;  // Element found, return index
        }
    }
    return -1;  // Element not found
}

Practice Idea: Modify the linear search algorithm to find all occurrences of a target value in an array, not just the first one. Return an array of indices where the target appears. For example, searching for 2 in [1, 2, 3, 2, 4, 2] should return [1, 3, 5].

Real-world Applications:

  • Small Datasets: When dealing with small collections where efficiency isn't critical.
  • Unsorted Data: When data isn't sorted and sorting would take more time than searching.
  • Simple Implementations: In scripts and simple programs where code clarity is more important than performance.
  • Education: As a teaching tool to introduce searching algorithms to beginners.
  • Database Querying: Sometimes used in database full-table scans when no index is available.

Summary: Linear Search is the most fundamental searching algorithm that every programmer should understand. While not efficient for large datasets with its O(n) time complexity, it's simple to implement, works on any data (sorted or unsorted), and serves as an excellent introduction to the concept of algorithm analysis. Understanding Linear Search provides a foundation for appreciating more efficient algorithms like Binary Search.
