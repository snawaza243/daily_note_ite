Selection Sort

Category: Sorting (Green)

Introduction: Selection Sort is a simple, intuitive sorting algorithm that works by repeatedly finding the minimum element from the unsorted portion and putting it at the beginning. Think of it like selecting the smallest card from a hand of playing cards and placing it in front, then repeating with the remaining cards. While not efficient for large datasets, it's easy to understand and has the advantage of making the minimum number of swaps (O(n) swaps), which can be beneficial when memory writes are expensive.

Working Principle: The algorithm maintains two subarrays: the sorted subarray and the unsorted subarray. In each iteration, it finds the minimum element from the unsorted subarray and swaps it with the first element of the unsorted subarray:

  1. Start with the entire array as unsorted.
  2. Find the minimum element in the unsorted subarray.
  3. Swap it with the first element of the unsorted subarray.
  4. Move the boundary between sorted and unsorted subarrays one element to the right.
  5. Repeat until the entire array is sorted.
Pseudocode:
selectionSort(arr):
  n = length(arr)
  for i from 0 to n-1:
    min_index = i
    for j from i+1 to n-1:
      if arr[j] < arr[min_index]:
        min_index = j
    swap arr[i] and arr[min_index]

Example & Dry Run:
Let's sort the array: [64, 25, 12, 22, 11]

  • Pass 1: Find min in [64,25,12,22,11] → min=11 at index 4 → Swap with index 0 → [11, 25, 12, 22, 64]
  • Pass 2: Find min in [25,12,22,64] → min=12 at index 2 → Swap with index 1 → [11, 12, 25, 22, 64]
  • Pass 3: Find min in [25,22,64] → min=22 at index 3 → Swap with index 2 → [11, 12, 22, 25, 64]
  • Pass 4: Find min in [25,64] → min=25 at index 3 → Already in position → [11, 12, 22, 25, 64]
  • Pass 5: Only one element left → Array is sorted
  • Result: Sorted array: [11, 12, 22, 25, 64]

Time and Space Complexity:

  • Best Case: O(n²) - Still makes all comparisons
  • Average Case: O(n²) - Quadratic time complexity
  • Worst Case: O(n²) - Reverse sorted or random data
  • Space Complexity: O(1) - In-place sorting, uses constant extra space
  • Swaps: O(n) - Makes exactly n-1 swaps in worst case

// C Code Example
#include <stdio.h>

void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

void selectionSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        int min_index = i;
        
        // Find the minimum element in unsorted array
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[min_index]) {
                min_index = j;
            }
        }
        
        // Swap the found minimum element with the first element
        swap(&arr[min_index], &arr[i]);
    }
}

// Selection sort that also finds maximum (dual selection)
void selectionSortDual(int arr[], int n) {
    int left = 0, right = n - 1;
    
    while (left < right) {
        int min_index = left;
        int max_index = right;
        
        // Find min and max in current range
        for (int i = left; i <= right; i++) {
            if (arr[i] < arr[min_index]) min_index = i;
            if (arr[i] > arr[max_index]) max_index = i;
        }
        
        // Swap min with left position
        swap(&arr[min_index], &arr[left]);
        
        // If max was at left, it's now at min_index
        if (max_index == left) {
            max_index = min_index;
        }
        
        // Swap max with right position
        swap(&arr[max_index], &arr[right]);
        
        left++;
        right--;
    }
}

// Stable selection sort (maintains relative order of equal elements)
void stableSelectionSort(vector& arr) {
    int n = arr.size();
    for (int i = 0; i < n - 1; i++) {
        int min_index = i;
        
        // Find minimum element
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[min_index]) {
                min_index = j;
            }
        }
        
        // Shift elements instead of swapping to maintain stability
        int min_value = arr[min_index];
        while (min_index > i) {
            arr[min_index] = arr[min_index - 1];
            min_index--;
        }
        arr[i] = min_value;
    }
}

def selection_sort(arr):
    """
    Sorts an array using selection sort algorithm.
    
    Args:
        arr: List of comparable elements
    
    Returns:
        The sorted list (in-place sort)
    """
    n = len(arr)
    for i in range(n - 1):
        min_index = i
        
        # Find the minimum element in unsorted array
        for j in range(i + 1, n):
            if arr[j] < arr[min_index]:
                min_index = j
        
        # Swap the found minimum element with the first element
        arr[i], arr[min_index] = arr[min_index], arr[i]
    
    return arr

function selectionSort(arr) {
    const n = arr.length;
    for (let i = 0; i < n - 1; i++) {
        let minIndex = i;

// Version with metrics
function selectionSortWithMetrics(arr) {
    let comparisons = 0;
    let swaps = 0;
    const n = arr.length;
    
    for (let i = 0; i < n - 1; i++) {
        let minIndex = i;
        
        for (let j = i + 1; j < n; j++) {
            comparisons++;
            if (arr[j] < arr[minIndex]) {
                minIndex = j;
            }
        }
        
        if (minIndex !== i) {
            swaps++;
            [arr[i], arr[minIndex]] = [arr[minIndex], arr[i]];
        }
    }
    
    return { sortedArray: arr, comparisons, swaps };
}

// Functional programming style (immutable)
function selectionSortImmutable(arr) {
    const sorted = [...arr];
    selectionSort(sorted);
    return sorted;
}

// Generic version with custom comparator
function selectionSortCustom(arr, compareFn = (a, b) => a - b) {
    const n = arr.length;
    for (let i = 0; i < n - 1; i++) {
        let extremeIndex = i;
        
        for (let j = i + 1; j < n; j++) {
            if (compareFn(arr[j], arr[extremeIndex]) < 0) {
                extremeIndex = j;
            }
        }
        
        if (extremeIndex !== i) {
            [arr[i], arr[extremeIndex]] = [arr[extremeIndex], arr[i]];
        }
    }
    return arr;
}

Real-world Applications:

  • Small Datasets: Useful for small arrays where simplicity is more important than efficiency
  • Memory-Constrained Systems: When memory writes are expensive, as it makes minimum swaps
  • Educational Tool: Excellent for teaching sorting algorithm concepts due to its simplicity
  • Hardware Sorting: Sometimes used in hardware implementations due to predictable behavior
  • Partially Sorted Data: Can be efficient when only a few elements are out of place
  • Embedded Systems: In systems with limited computational resources for small datasets
  • Algorithm Building Block: Used as a component in more complex algorithms

Summary: Selection Sort is a fundamental sorting algorithm that demonstrates the basic principles of comparison-based sorting. While its O(n²) time complexity makes it inefficient for large datasets, its simplicity, predictable behavior, and minimal number of swaps make it suitable for specific use cases. Understanding Selection Sort provides a solid foundation for learning more advanced sorting algorithms and helps develop intuition about algorithm analysis and optimization.
