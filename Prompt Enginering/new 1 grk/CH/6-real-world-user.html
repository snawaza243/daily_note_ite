Real-World Usage Examples of LLMs

This module explores practical applications of Large Language Models (LLMs) across various domains. You'll learn how to craft effective prompts for common real-world tasks, with examples and interactive demonstrations.

1. Structured Data Extraction

LLMs can transform unstructured text into structured formats like JSON, CSV, or tables. This is particularly useful for data processing pipelines.

Example Prompt:

Extract the product details from the following text as JSON with fields for name, price, and features:

"The Quantum X900 laptop features a 15.6" 4K display, 32GB RAM, and 1TB SSD storage. It's priced at $1,299.99 and comes with a 2-year warranty."

Output:

{
  "name": "Quantum X900 laptop",
  "price": 1299.99,
  "features": [
    "15.6\" 4K display",
    "32GB RAM",
    "1TB SSD storage",
    "2-year warranty"
  ]
}

Python Implementation:

from transformers import pipeline

# Initialize the text-to-json pipeline
extractor = pipeline("text2text-generation", model="google/flan-t5-large")

# Define the prompt and input text
prompt = "Extract the product details as JSON with name, price, and features:"
text = "The Quantum X900 laptop features a 15.6\" 4K display..."

# Generate the structured output
result = extractor(prompt + text)
print(result['generated_text'])

4. Coding Assistance

LLMs excel at explaining, generating, and debugging code across multiple programming languages.

Code Generation Prompt:

Write a Python function that takes a list of numbers and returns a dictionary with the mean, median, and mode. Include docstring and type hints.

Output:

from typing import List, Dict
from statistics import mean, median, mode

def calculate_stats(numbers: List[float]) -> Dict[str, float]:
  """Calculate basic statistics from a list of numbers.

  Args:
    numbers: List of numeric values

  Returns:
    Dictionary with 'mean', 'median', and 'mode' keys
  """
  try:
    mode_val = mode(numbers)
  except StatisticsError:
    mode_val = None

  return {
    "mean": mean(numbers),
    "median": median(numbers),
    "mode": mode_val
  }

Debugging Prompt:

Explain the error in this Python code and suggest a fix:

def calculate_discount(price, discount):
  return price - (price * discount)

# When called with calculate_discount(100, 0.2) it works,
# but with calculate_discount(100, 20) it gives wrong result

Output:

# The issue is inconsistent discount parameter handling.
# The function expects discount as decimal (0.2 for 20%),
# but sometimes gets percentage (20 for 20%).

# Solution 1: Document expectation and validate input
def calculate_discount(price: float, discount: float) -> float:
  """Calculate price after discount.
  Args:
    discount: Decimal (e.g., 0.2 for 20% off)
  """
  if discount > 1: # Assume percentage was passed
    discount /= 100
  return price * (1 - discount)

Imagine spinning a coinâ€”while it's spinning, it's not just heads or tails, but in a mix of both states. Qubits work similarly until you "look" at them (measure them), when they settle into one state.

Entanglement is like having two magical coins that are linkedâ€”no matter how far apart they are, if you spin one and it lands heads, the other will instantly land tails. This spooky connection helps quantum computers solve certain problems much faster than regular computers.

6. Designing Chatbots

LLMs can power conversational agents with predefined behaviors and personalities.

Chatbot Prompt:

You are a friendly customer support bot for an e-commerce store called "TechGadgets". Your role is to:
- Greet customers warmly
- Answer questions about products, shipping, and returns
- Escalate complex issues to human agents
- Maintain a helpful but professional tone

The customer says: "Hi, I ordered a wireless charger 3 days ago but haven't received any shipping update. Can you help?"

Output:

"Hello! Thanks for reaching out to TechGadgets support. I'd be happy to check on your order. Could you please share your order number so I can look up the details? Our standard shipping usually takes 3-5 business days, but I can check the current status for you."

Python Implementation:

from transformers import pipeline

# Initialize the chatbot with our system prompt
chatbot = pipeline("conversational", model="facebook/blenderbot-400M-distill")

# System message defines the bot's behavior
system_prompt = """You are a friendly customer support bot for TechGadgets..."""

# Conversation history
conversation = [
  {"role": "system", "content": system_prompt},
  {"role": "user", "content": "Hi, I ordered a wireless charger..."}
]

# Generate response
response = chatbot(conversation)
print(response['generated_text'])

9. Prompt Engineering Workflow

Effective prompt engineering follows an iterative process of refinement and testing.

Best Practices:

  • Start specific: Clearly define the task and desired output format
  • Provide examples: Show the model what good output looks like
  • Use constraints: Limit response length or format when needed
  • Iterate: Test multiple prompt variations to find what works best
  • Evaluate: Check outputs for accuracy, completeness, and relevance

Prompt Refinement Example:

Initial Prompt:

Tell me about machine learning

Refined Prompt:

Explain supervised machine learning to a business executive in 3 paragraphs. Focus on practical applications in customer analytics and use analogies from everyday life. Avoid mathematical terminology.

Further Refined:

You are explaining supervised machine learning to a retail company's CEO who has no technical background. Structure your response as:
1) Core concept analogy (like teaching a child with examples)
2) Two business use cases from their industry (customer segmentation and demand forecasting)
3) Implementation requirements (data needed, timeline, expected outcomes)
Keep each section under 100 words.
